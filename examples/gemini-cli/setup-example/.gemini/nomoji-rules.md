EMOJI USAGE RULES:

DOCUMENTATION & MARKDOWN:
- STRICT: Do not use emojis in markdown files, README files, or documentation.
- This includes headers, lists, paragraphs, and code examples.
- Use clear, professional language without emoji decoration.

CONSOLE OUTPUT:
- STRICT: Do not use emojis in console.log, console.error, console.warn, or any console output.
- Use plain text for all terminal output.

CLI TOOLS & COMMAND-LINE OUTPUT:
- STRICT: Do not use emojis in CLI tool output, progress bars, spinners, or command-line messages.
- Use ASCII characters and plain text only.

LOGGING & ERROR MESSAGES:
- STRICT: Do not use emojis in application logs, error messages, debug output, or logging statements.
- Logs should be machine-parseable and professional.

CODE COMMENTS:
- MODERATE: Avoid using emojis in code comments.
- Write clear, descriptive comments using words only.

GIT COMMIT MESSAGES:
- STRICT: Do not use emojis in git commit messages.
- Use conventional commit format with plain text.

REMEMBER: Professional code should prioritize clarity and consistency over decoration.
When in doubt, do not use emojis.
